## 什么是SBTI？

全称 Silly Big Personality Test，翻译过来就是「傻大个儿人格测试」。

它以经典的MBTI为框架恶搞改编，把正经的16型人格变成了27种荒诞自嘲标签——

死者、吗喽、狗屎人、草者、ATM-er、幽灵、机器人……

没有心理学依据，纯属娱乐玩梗。但年轻人就是爱玩，因为每个标签都像在说自己。

## 我用WorkBuddy做了什么？

整个过程就4步：

### 第一步：搜集素材

先让WorkBuddy搜索SBTI的完整题目和27种人格定义。网上有好几个版本，我选了最完整的一个，整理成结构化数据——15道题、4个维度、27种结果类型。

这一步如果手动做，至少要半小时。AI几秒钟搞定。

### 第二步：写前端代码

把整理好的数据交给WorkBuddy，让它写HTML+CSS+JS。

核心逻辑包括：答题计分、四维度计算（E/I、S/N、T/F、J/P）、结果匹配排序、匹配度百分比计算、十五维度评分展示。

全部在前端完成，不需要后端服务器。

### 第三步：画卡通头像

这是最费心思的部分。每种人格需要一个专属头像，风格要统一。

我用SVG手写了27个简约卡通头像——死者是灰色的X眼表情、吗喽是猴子头、ATM-er是绿色机器造型……每个都有辨识度。

### 第四步：加动效和打磨

粒子背景动画、渐变配色、按钮悬停效果、数字滚动动画、进度条动画……这些细节让页面看起来不那么「程序员审美」。

最终效果：深色背景+紫色渐变主题+浮动粒子，打开就有科技感。

## 开发过程中的调试

你以为做完这四步就完了？没那么简单。

测试功能做完后，我又加了一个"保存图片"功能——让用户能把结果保存到手机相册。但就是这个功能，花了我将近一个小时排查bug。

**问题表现**：页面显示一切正常，但保存出来的图片内容不对：

- 页面显示15个维度评分 → 图片里只有8个
- 页面有专属头像 → 图片里头像消失了
- 页面有人格解读方块 → 图片里完全没有

数据是对的，但图片就是不对。

### 第一步：用PIL扫描图片定位问题

我没有直接盯着代码看，而是先用Python PIL库逐行扫描生成的图片，检查哪些像素是紫色的进度条。

结果发现一个规律：**所有紫色进度条都挤在左列，右列完全空的**。

而且维度5的进度条宽度是632像素，比列宽还大——这说明进度条绘制超出了它的列边界。

### 第二步：找到根因——像素单位混淆

代码里用了Canvas缩放：

```javascript
const SCALE = 2;
ctx.scale(SCALE, SCALE);  // 把坐标系放大2倍
```

然后计算列宽时，单位用混了：

```javascript
// 修复前：INNER是Canvas像素，但colW被当CSS像素用了
const colW = (INNER - 32 - 20) / 2;
const dx = PAD + 16 + col * (colW + gap);
```

在`ctx.scale(2, 2)`生效后，这个值被放大到了4倍，导致右列的绘制坐标超出了画布边界，直接被裁剪掉了。

修复方案是**统一使用CSS像素**（在scale之后会被自动放大）：

```javascript
// 修复后：先转换成CSS像素
const CWcss = CW / SCALE, PADcss = PAD / SCALE;
const colWcss = (CWcss - 2 * PADcss - 20) / 2;
const dx = PADcss + 16 + col * (colWcss + gap);
```

### 第三步：还有两个小坑

**坑1：头像SVG解析错误**

页面显示有头像，但保存的图片没有。原因是外部库返回的SVG字符串可能已经带了`<svg>`标签，我又在外面包了一层，导致嵌套了两层SVG，浏览器解析失败。

**坑2：人格解读方块完全缺失**

这个纯粹是漏写——`drawResultCard`函数里根本没有绘制这个元素。加上就解决了。

### 调试心得

1. **图片比代码更诚实**：代码里写了"绘制15个维度"，但实际只有8个。用PIL扫描图片能直观看到实际绘制了什么。

2. **Canvas的scale()是陷阱**：在Canvas里用`ctx.scale()`后，所有后续坐标计算都要注意单位。简单原则：统一用CSS像素。

3. **让AI帮你排查**：整个调试过程，我只描述问题，WorkBuddy完成了读代码、定位问题、找根因、出方案的全部工作。

## 部署上线

做完之后不能只存在本地，要让别人也能访问。

### 第一步：上传到GitHub

整个项目就是一个HTML文件+几张图片，不需要复杂的后端。我把它上传到GitHub仓库：

```bash
# 初始化git仓库
cd sbti-test
git init
git add index.html sbti-article.md *.png
git commit -m "SBTI测试网站完成"
git remote add origin https://github.com/zgedu-new/panda-bamboo-game.git
git push -u origin main
```

### 第二步：配置GitHub Pages

在GitHub仓库的Settings → Pages选项中：

1. Source 选择 **Deploy from a branch**
2. Branch 选择 **main**
3. Folder 选择 **/ (root)** 或 **/sbti-test**（取决于你的目录结构）

点击Save，等待1-2分钟，网站就上线了。

### 第三步：绑定自定义域名（可选）

如果你有自己的域名，可以在Custom domain里填入域名地址，开启HTTPS。

### 最终效果

部署完成后，用户可以直接访问：
```
https://zgedu-new.github.io/panda-bamboo-game/sbti-test/
```

完全免费，不需要买服务器。
